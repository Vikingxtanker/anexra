# Step 3 — Create Database Structure

Go to:

SQL Editor → New Query

Run this:

create table profiles (
id uuid references auth.users on delete cascade primary key,
full_name text,
email text unique,
role text default 'student',
college_name text,
degree text,
semester text,
created_at timestamp with time zone default timezone('utc', now())
);

alter table profiles enable row level security;

This becomes your master user table.

# Step 4 — Auto Create Student Profile After Signup

Create trigger:

create function public.handle_new_user()
returns trigger
set search_path = public
as $$
begin
insert into public.profiles (
id,
email
)
values (
new.id,
new.email
);

return new;
end;
$$ language plpgsql security definer;

create trigger on_auth_user_created
after insert on auth.users
for each row execute procedure public.handle_new_user();

Now every signup automatically creates:

* student profile
* linked account

# Step 8 — Setup Row Level Security (RLS)

## Profiles Policy

Allow users to see only their own profile.

Run:

create policy "Users can view own profile"
on profiles
for select
using (auth.uid() = id);

create policy "Users can update own profile"
on profiles
for update
using (auth.uid() = id);

---

# Step 9 — Notes Access Policy

Initially allow authenticated students.

create policy "Authenticated users can view notes"
on notes
for select
to authenticated
using (true);

Later you can restrict:

* by semester

* by subscription

# Important Recommendation

Do NOT build:

* separate database for students
* separate auth system later

Keep everything unified from day 1.

Your future structure should eventually become:

```txt id="x86v7m"
/student/*
/doctor/*
/patient/*
/admin/*
```

all powered by:

* one auth system
* one profiles table
* role-based permissions

That is proper SaaS architecture.
